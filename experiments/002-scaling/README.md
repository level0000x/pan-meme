# 实验二：数据量缩放实验

**日期**: 2026-07-08

## 目的

验证 H₁ 收敛性：数据量越大，收敛越稳定。

## 设计

同一领域（演化生物学），四个规模，独立运行。参数统一为 `t_max=20.0, max_steps=80000, auto-optimize`。

| 规模 | 字符数 | 文件 |
|------|--------|------|
| S1 | ~400 | `inputs/scale_1.txt` |
| S2 | ~1200 | `inputs/scale_2.txt` |
| S3 | ~3100 | `inputs/scale_3.txt` |
| S4 | ~4900 | `inputs/scale_4.txt` |

## 结论

### 正面发现

1. **收敛率 100%**：四个规模全部收敛，H₁ 在 t_max=20.0 下成立
2. **收敛时间恒定**：收敛时间 t≈11 不随数据量变化，说明 ODE 动力学由结构决定而非规模
3. **终态稳定**：终态 D→1, B→0, S→1 在所有规模下完全一致
4. **H₃ 可逆性**：100% 词恢复率在所有规模成立

### 局限

1. **模因数恒为 2**：Louvain 社区检测在当前参数下始终将图分为 2 个模因，无法验证"更多数据 → 更多模因"的预测
2. **原型单一**：所有规模都收敛到 Stone 原型，无法验证原型多样性
3. **规模跨度不够**：4 个规模都在 400-5000 字符范围内，未达到 Wikipedia 级别的 10⁵ token
4. **性能瓶颈**：S4 的 Phase 1 耗时 6 分钟，Warshall O(n³) 在 4000 节点上已接近笔记本极限

### 对照验收标准

| 等级 | 条件 | 是否满足 |
|------|------|---------|
| 证伪 | 收敛率 < 50% | 否 |
| 部分验证 | 收敛率 > 50% 但 < 90% | 否 |
| **验证** | 收敛率 > 90% | **是**（100%） |
| 强验证 | 满足"验证" + 每领域 ≥ 3 个原型 | 否 |
