# 实验 010：缺失原型补全

**日期**：2026-07-08  
**版本**：v4.3  
**状态**：✅ 完成 — 4/4 仍缺失（阴性结果，具诊断意义）

---

## 一、问题背景

实验 007/008 在 4 领域 × 3 分辨率测试中观测到 3-4 种原型：Source、StableCore、Transient、Stone。但理论上定义的九类原型中，仍有四种从未被观测到：

- **Decay**（衰减型）— D↓, B↓, ρ↓, R→, S↓
- **Oscillatory**（振荡型）— 五维周期性波动
- **Resilient**（韧态）— D→, B↓, ρ↓, R→, S↑↑
- **Sink**（汇型）— D→, B↓, ρ↑, R↓, S→

## 二、假设

> **H₅（数据规模假设）**：使用更大规模的输入（完整 Wikipedia 文章，而非 1000 字摘要），产生的 CW 复形结构更丰富，社区分化更显著，从而可能触发缺失的演化原型。

即：**限制因素不是分类器，而是输入数据的信息量。**

## 三、实验设计

### 3.1 数据

| 类别 | 数量 | 平均字数 | 预期原型 |
|------|------|----------|----------|
| Decay | 7 | 47,735 | 衰减/消亡的技术/产品 |
| Oscillatory | 8 | 32,169 | 周期性自然/社会现象 |
| Resilient | 10 | 75,353 | 经历冲击后长期存续的事物 |
| Sink | 8 | 64,696 | 信息汇聚但低扩散的系统 |

**总计**：33 篇 Wikipedia 全文，1,871,956 字。对比实验 009 的摘要（平均 ~2000 字/篇），信息量提升了约 **28 倍**。

### 3.2 方法

```
Phase 1-5 完整管线（γ=2.0, t=25）
  → 33 个概念 × 每个 16-24 个社区 = 649 个独立 ODE 轨迹
  → 每个轨迹通过 classifier 分类为 9 种原型之一
  → 统计各原型出现频率
```

## 五、失败分析：为什么缺失原型无法触发？

### 5.1 Decay：D 的天然单调性

分类器中 Decay 的两个触发条件：

```
条件 1: r_decay > 0.3 && r_init > 0.5 && d_decay > 0.2 && d_final < 0.5
条件 2: d_decay > 0.3 && d_peak > 0.5 && d_peak_t < t_max/2
```

两个条件都要求 **D 先上升后下降超过 0.2**。但 v4.2 的参数设计（δ₃ 立方映射 0.08-0.97）使 D 天然趋向于 1.0——α₂（S→D 深化）对所有社区都显著，而抑制 D 的力量仅来自 δ₁ 和 δ₃，二者均作用于 R，而非 D 自身。**D 的方程中没有直接的衰减项**：

```
dD/dt = -α₁·R·D + α₂·S·(1-D)
```

当 R 低时，-α₁·R·D ≈ 0，只剩下 +α₂·S·(1-D) > 0，D 必然上升。D 不可能在没有任何反向驱动力的情况下衰减。

**修复方向**：引入 D 的自衰减项，如 `-α₃·Φ_rho(ρ)·D`（信息饱和后的自发衰退）。

### 5.2 Oscillatory：自治 ODE 的无振荡性

触发条件：`oscillation_count >= 8`（8 次符号变化）。

当前 ODE 系统是**一阶自治系统**（参数不显含 t），五维相空间中所有轨迹单调趋近于稳定固定点。在 Lipschitz 连续条件下，自治系统不可能产生真正的极限环——至多因数值噪声产生 1-2 次微小符号变化。

**修复方向**：引入周期性外部驱动力 I_ext(t) = sin(ωt) 或显含时间的参数，使系统变为非自治。

### 5.3 Resilient：S 与 D 的同步收敛

触发条件：
```
条件 1 (near_converged): s_min > 0.7 && s_final > d_final + 0.05
条件 2 (early): s_final > 0.85 && s_min > 0.7 && d_final < 0.85 && d_final > 0.3
```

条件 1 要求收敛后 S 领先 D；条件 2 要求 S 已近收敛但 D 落在 0.3-0.85 区间。在当前参数下，α₂ 和 ε₁ 映射方式不同（立方 vs 线性），但都随社区大小单调——大社区中两者同时大，小社区中两者同时小。S 和 D 始终同步运动，无法产生"S 高而 D 低"的状态。

**修复方向**：为 ε₁（D→S 转化）添加独立于 α₂ 的扰动项，或在参数空间中引入反相关的社区对。

### 5.4 Sink：ρ-D/B 的耦合收敛

触发条件：`rho_final > 0.6 && d_final < 0.4 && b_final < 0.4`

要求最终状态中 ρ 高而 D、B 低。但在当前方程中：

```
dρ/dt = -γ₁·R·ρ + γ₂·(1-ρ)·I_ext
dB/dt = β₁·R·(1-B) - β₂·D·B
dD/dt = -α₁·R·D + α₂·S·(1-D)
```

ρ 和 B/D 通过 R 耦合——当 R 收敛到低值时，ρ 也趋向低值（除非 I_ext 异常大）。实际运行中 R → 0 会使所有维度收敛，ρ 不可能独立保持高位。

**修复方向**：引入 ρ 的独立维持项，如 `+γ₃·D·(1-R)`——即使 R 低，高 D 的结构也能维持 ρ。

## 六、结论

### 6.1 核心发现

1. **H₅ 被证伪**：数据规模扩大 28 倍不能产生新的原型类别。限制因素在 ODE 动力学本身，而非输入数据。
2. **参数均匀化是根本原因**：v4.2 的参数设计（δ₃ 立方映射等）确保了所有社区向固定点收敛——这是实验 008 成功的原因，也是缺失原型无法出现的原因。
3. **四种缺失原型各自有明确的动力学障碍**：
   - Decay → D 无自衰减项
   - Oscillatory → 自治系统无极限环
   - Resilient → S/D 参数耦合过强
   - Sink → ρ-S/D 通过 R 的耦合无法打破

### 6.2 下一步

缺失原型的补全不需要更多数据实验，而需要 **v4.4 动力学重构**：

| 原型 | 需要的动力学修改 | 优先级 |
|------|-----------------|--------|
| Decay | D 自衰减项 `-α₃·Φ_rho(ρ)·D` | P0 |
| Sink | ρ 独立维持项 `+γ₃·D·(1-R)` | P0 |
| Resilient | ε₁ 与 α₂ 反相关参数空间 | P1 |
| Oscillatory | 非自治外部驱动 I_ext(t) | P2 |

### 6.3 对理论的贡献

这个阴性结果本身有重要理论价值：它确证了 v4.2 动力学是**高度单调的、指向单一固定点的系统**——这对于"收敛性"（实验 008 的核心指标）是优势，但对于"多样性"（九类原型描述的丰富动力学）是限制。完整的泛模因理论需要在这两者之间找到平衡。
