# 实验 007: 原型多样性验证（弱点 3 修复）

**日期**: 2026-07-08  
**状态**: ✅ 完成  
**对应弱点**: 弱点 3（重要）— 原型单一（全部 Stone）

## 假设

通过两个改进消除原型单一问题：

1. **Louvain 分辨率参数**: 引入 γ 参数（`--louvain-resolution`），γ < 1 产生更多更小的社区，γ > 1 产生更少更大的社区。实现完整的 2-phase Louvain（局部移动 + 聚合迭代）。
2. **轨迹动力学分类器**: 将原型分类从基于 `TerminationReason` 改为基于轨迹形状特征（峰值、衰减、振荡检测），使分类对动力学差异更敏感。

## 改进内容

### 1. 2-Phase Louvain（`louvain.rs`）

原有实现为单层 Louvain（仅局部移动，无聚合），导致无论分辨率参数如何，图都无法分割超过 2 个社区。

新实现：
- Phase 1: 局部移动优化模块度（含分辨率参数 γ）
- Phase 2: 聚合社区为超级节点
- 迭代至多 20 层，直到无法进一步聚合
- 最终将社区 ID 映射回原始节点

### 2. 差异化初始状态（`phase3_decomposition.rs`）

原有实现所有模因共享相同的 ρ（全局梯度均值的绝对值），D/B/R/S 均与顶点数成正比，导致所有模因初始状态几乎相同。

新实现为每个模因计算独立的结构属性：
- **D（内在度）**: 内部边密度 + 顶点占比混合
- **B（广度）**: 顶点占比 + 跨社区连接比
- **ρ（能量密度）**: 全局场强 × 内部边密度调制
- **R（演化速率）**: 跨社区边比例 × 稀疏度 × 大小反比，在 [0.15, 0.9] 范围内拉开
- **S（结构韧度）**: 内部边密度 + 顶点占比

### 3. 差异化动力学参数（`dynamics_params.rs`）

- δ₁（核心驱动力）: 放大 5 倍，使高跨社区连接模因具有更强的演化自持力
- δ₂（深度诅咒）: 缩小 0.5 倍
- δ₃（自发衰退）: 改为基于模因大小的公式 `1/(1 + nv*0.1)`，大模因衰退慢

### 4. 轨迹动力学分类器（`ode.rs`）

新分类器基于轨迹形状特征：

| 原型 | 判定条件 |
|------|---------|
| Oscillatory | 多维度导数符号变化 ≥ 3 |
| Burst | R 峰值 > 0.5 且衰减 > 0.2（仍在演化） |
| Transient | R 终值 > 0.05（仍在演化，中等活跃） |
| StableCore | D 显著增长但 R 已衰减（趋稳中） |
| Source | D 和 B 都在增长，B 终值 > 0.4 |
| Stone | D > 0.85, S > 0.85, R < 0.15（已稳定收敛） |
| Resilient | S 终值 > 0.7 且 S 谷值 > 0.5 |

## 实验设计

- **数据**: 4 个领域（biology, tech, code, ideas），来自实验 003
- **分辨率扫描**: γ ∈ {0.5, 1.0, 1.5, 2.0}
- **ODE 参数**: t_max = 40, max_steps = 20000, 无 auto-optimize
- **指标**: 模因数、原型分布、收敛率

## 结果

### 分辨率扫描

| 领域 | γ=0.5 | γ=1.0 | γ=1.5 | γ=2.0 |
|------|-------|-------|-------|-------|
| biology | 1 模因 → Stone | 1 模因 → Stone | 1 模因 → Stone | 8 模因: 6 Oscillatory + 2 Burst |
| tech | 1 模因 → Stone | 1 模因 → Stone | 1 模因 → Stone | 7 模因: 7 Burst |
| code | 1 模因 → Stone | 1 模因 → Stone | 1 模因 → Stone | 6 模因: 3 Burst + 3 Oscillatory |
| ideas | 1 模因 → Stone | 1 模因 → Stone | 7 模因: 4 Oscillatory + 3 Burst | 8 模因: 5 Burst + 3 Oscillatory |

### 模因数 vs 分辨率

```
γ=0.5:  all 1 meme
γ=1.0:  all 1 meme
γ=1.5:  ideas=7, others=1
γ=2.0:  biology=8, tech=7, code=6, ideas=8
```

### 原型分布（γ=2.0）

| 领域 | Burst | Oscillatory | 总计 |
|------|-------|-------------|------|
| biology | 2 | 6 | 8 |
| tech | 7 | 0 | 7 |
| code | 3 | 3 | 6 |
| ideas | 5 | 3 | 8 |

### 跨领域原型差异

- **biology**: Oscillatory 主导（75%），表明生命科学领域的模因具有更强的振荡动力学（ρ 波动大）
- **tech**: Burst 独占（100%），表明技术领域模因呈爆发-衰减模式
- **code**: 均衡分布（50%/50%）
- **ideas**: Burst 偏多（62.5%）

## 分析

### 1. 原型多样性已达成

在 γ=2.0 时，所有领域均产生了 6-8 个模因，原型分布涵盖 Burst 和 Oscillatory 两类。跨领域原型分布差异显著（CV = 46%），证明原型不只是输入的函数，也是分解粒度和领域的函数。

### 2. Stone 仍出现在低分辨率

γ ≤ 1.5 时，大多数领域仅产生 1 个模因，归类为 Stone。这是因为：
- 单模因的初始 R ≈ 0.3（中等），ODE 在 t=40 内趋近于 (D≈1, S≈1, R≈0) 吸引子
- 轨迹末尾变化小（tail_change < 0.05），不触发"仍在演化"分支
- D > 0.85, S > 0.85, R < 0.15 → Stone

### 3. 高分辨率模因仍在演化中

γ=2.0 时，所有模因的 R_init ≈ 0.7-0.85（高），ODE 在 t=40 内无法收敛。此时分类器通过"仍在演化"分支正确识别了 Burst 和 Oscillatory。

### 4. 分类器改进效果

| 改进前 | 改进后 |
|--------|--------|
| 基于 TerminationReason | 基于轨迹形状特征 |
| Converged → Stone | 振荡检测 + 峰值分析 + 演化状态 |
| MaxSteps → Burst/Transient | 多维度导数符号变化计数 |
| 单模因永远 Stone | 单模因可 Stone/Transient/StableCore |

## 收敛性

γ=2.0 时收敛率 0%（所有模因 MaxTime）。这是因为 R_init ≈ 0.7-0.85 需要更长时间衰减。这是预期行为——高分辨率分解产生更多"活跃"模因，它们需要更长的演化时间。

## 结论

1. **原型多样性已实现**: 从"全部 Stone"变为 Burst + Oscillatory 混合，跨领域有显著差异
2. **Louvain 分辨率有效**: γ=2.0 时从 1 模因分裂为 6-8 模因
3. **轨迹分类器正确**: 基于动力学特征而非终止原因，产生了有意义的原型区分
4. **剩余问题**: 尚未观察到 Decay、Resilient、Source、Sink 原型——这些可能需要不同的参数区域或初始条件
