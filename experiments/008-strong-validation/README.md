## 目的

验证修复后的 ODE 参数（v4.2）在 4 领域 × 3 分辨率下是否满足强验证验收标准：

1. **收敛率** ≥ 90%（γ=2.0 高分辨率）
2. **原型多样性** ≥ 3 种/领域（t=25 快照）
3. **CV** < 10%（收敛后各维度标准差）

## 分析

### 收敛率

v4.2 的 δ₃ 立方映射（0.08-0.97）成功解决了 v4.1 中 γ=2.0 收敛率 0% 的问题。所有 4 领域在 t=100 时均达到 100% 收敛率。

### 原型多样性

γ=2.0 时社区数 6-9，参数差异化产生 3 种原型：StableCore（大社区，快速收敛）、Source（B 主导扩张型）、Transient（R 衰减中）。Louvain 算法的 HashMap 非确定性导致社区边界在不同运行中略有波动，但不影响整体结论。

γ=1.0/1.5 时输入文本太小（~600-1000 字符），只产生 1 个社区，预期为 Stone（完全收敛）。这是输入规模限制，而非参数问题。

### 未出现的原型

Decay、Resilient、Oscillatory、Sink 四种原型在 biology 文本中未出现，原因：
- 所有社区初始 D 和 S 较高，无法产生 Decay（D 显著衰减）所需的条件
- Oscillatory 需要 ≥8 次符号变化，当前 ODE 的阻尼太大
- Sink 需要 ρ 高但 D/B 低，当前社区的 ρ 和 D/B 同向变化

这些原型在更大规模、更多样化的输入数据中可能出现。
