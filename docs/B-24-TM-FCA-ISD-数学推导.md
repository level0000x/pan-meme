## §1. 问题陈述

### 1.1 目标

用信息-结构动力学（ISD）框架判定一台未解决的 BB(6) 候选机（Antihydra）的停机行为，
不依赖步数枚举、不依赖 Collatz 猜想的数论分析、不依赖 ZFC 在无限集合上的枚举能力。

### 1.2 核心假设

存在一个映射 Φ：图灵机 M → FCA 概念格 B(M)，使得 B(M) 上的 N 算子谱半径
ρ(J) 能够区分 M 的停机/非停机行为：

- ρ_∞(M) = 0 ⟹ M 最终停机
- ρ_∞(M) > 0 ⟹ M 永不停机

## §2. 形式化定义

### 2.1 图灵机

**定义 2.1**（图灵机）。一台 2-符号图灵机为五元组 M = (Q, Γ, δ, q₀, F)，其中：

- Q = {0, 1, …, n-1} 为有限状态集（n 个状态）
- Γ = {0, 1} 为带符号集
- δ: Q × Γ → Q ∪ {n} × Γ × {-1, 0, 1} 为转移函数（n 表示停机状态）
- q₀ = 0 为初始状态
- F = {n} 为接受状态集（停机）

运行从全零带开始，读写头在原点。

### 2.2 瞬时描述

**定义 2.2**（完整瞬时描述）。M 在时刻 t 的完整瞬时描述为：

```
ID_M(t) = (q(t), …, γ_{-1}(t), γ₀(t), γ₁(t), …, h(t))
```

其中 h(t) ∈ ℤ 为读写头位置。

**定义 2.3**（窗口化瞬时描述）。对窗口长度 L ∈ ℕ⁺，定义窗口化瞬时描述为：

```
ID_{M,L}(t) = (q(t), γ_{-L}(t), …, γ_L(t), h(t))
```

其中 h(t) ∈ [-L, L]。窗口捕获读写头周围 2L+1 个带单元。

### 2.3 状态转移图

**定义 2.4**（配置图）。M 的配置图为有向图 G_M = (V_M, E_M)，其中：

- V_M = {ID_M(t) : t ∈ ℕ}（所有可达配置）
- E_M = {(ID_M(t), ID_M(t+1)) : t ∈ ℕ}（单步转移）

**定义 2.5**（窗口化配置图）。M 在窗口 L 内的配置图为 G_{M,L} = (V_{M,L}, E_{M,L})，
其中 V_{M,L} 为 ID_{M,L}(t) 的投影，E_{M,L} 为对应的转移。

**关键性质**：
- 若 M 停机，则 G_M 为有限图（配置数 ≤ 停机步数）
- 若 M 不停机，则 G_M 为无限图（可能）

### 2.4 形式背景与 FCA 格

**定义 2.6**（形式背景）。形式背景为三元组 K = (G, M, I)，其中：

- G 为对象集
- M 为属性集
- I ⊆ G × M 为关联关系

**定义 2.7**（概念格）。K 的概念格 B(K) 为所有形式概念 (A, B) 的集合，其中：

- A ⊆ G, B ⊆ M
- A↑ = {m ∈ M : ∀g ∈ A, gIm}（A 的公共属性）
- B↓ = {g ∈ G : ∀m ∈ B, gIm}（B 的公共对象）
- A↑ = B 且 B↓ = A

偏序： (A₁, B₁) ≤ (A₂, B₂) ⟺ A₁ ⊆ A₂（等价于 B₂ ⊆ B₁）。

### 2.5 N 算子与谱半径

**定义 2.8**（N 算子）。N 算子为 5 维动力学系统，作用于概念格 B 的每个概念上：

```
x_i^{k+1} = N(x_i^k; {x_j^k : j ∈ N(i)})
```

其中 N(i) 为概念 i 在 Hasse 图中的邻居。

**定义 2.9**（Jacobian 谱半径）。在不动点 x* 处，N 算子的 Jacobian 为 J = D · W，
其中 D 为对角阵，W 为权重阵。谱半径 ρ(J) = max{|λ| : λ 为 J 的特征值}。

**定理 2.10**（v3.80 定理 6.15）。对任意有限概念格 B，ρ(J(B)) < 1。
即 N 迭代在有限格上总是收敛。

## §3. 理论提案：TM → FCA 格的映射

### 3.1 原始构造（v1）

**定义 3.1**（形式背景 K_{M,L}，v1）。对图灵机 M 和窗口 L，构造：

```
G = (Q × Γ × [-L, L]) ∪ Γ
M = {pos_p : p ∈ [-L, L]} ∪ {trans_{q→q'} : δ 中存在 q → q'}
I：(q, γ, p) ↔ pos_p
    (q, γ, p) ↔ trans_{q→q'}  iff δ(q, γ) = (q', γ', d)
```

**引理 3.2**（格嵌入，主张）。K_{M,L} 的概念格 B_{M,L} 的 Hasse 图同构于 M 在窗口 L
内的状态转移子图的传递闭包。

### 3.2 实验验证（v1 失败）

**实验 1**：对 5 台机器（SimpleHalter, SimpleNonHalter, BB5Champion, CurrentChampion, Antihydra）
运行 v1 构造。

**结果**：

| 机器 | 格概念数 | 格边数 | ρ_top |
|------|---------|--------|-------|
| 所有 5 台 | 2 | 1 | 0.54891053 |

**诊断**：形式背景 K_{M,L} 产生退化格（仅 2 个概念：顶和底）。原因：

1. 所有 (q, γ, p) 三元组都关联了 pos_p 属性 → 每个三元组有唯一的 pos_p
2. 对于相同的 (q, γ) 对，所有三元组共享相同的 trans_{q→q'} 属性
3. 关联矩阵过于"密集"，导致 FCA 概念仅包含顶（所有对象，无共享属性）和底（无对象，所有属性）

**结论**：v1 构造无效。形式背景不区分不同 TM，格结构对所有 TM 相同。

### 3.3 修正构造（v2：基于计算轨迹）

**定义 3.3**（形式背景 K_{M,L}，v2）。改为基于 M 的实际计算轨迹：

1. 模拟 M 运行 N 步，收集配置序列 {ID_{M,L}(t) : t = 0, …, N}
2. 采样（每 sample_every 步取一个配置）得到对象集
3. 属性分为四层：
   - **层 1**：分箱位置-符号属性（pos_even_0, pos_even_1, pos_odd_0, pos_odd_1）
   - **层 2**：全局共享属性（has_1s, all_0s, ones_lt_25, ones_25_50, ones_50_75, ones_gt_75, head_ctr, head_edge）
   - **层 3**：状态组属性（st_has_halt, st_self_loop, st_halt_adj）
   - **层 4**：转移类型属性（tr_{q→q'}）

**设计意图**：层 2 的全局属性被多个对象共享，创建重叠外延，迫使格结构更丰富。

### 3.4 实验验证（v2 部分成功）

**实验 2**：对 5 台机器运行 v2 构造，max_steps=50000, sample_every=2, 25k 对象。

**结果**：

| 机器 | 格概念数 | 格边数 | ρ_top |
|------|---------|--------|-------|
| SimpleHalter | 3 | 2 | 0.54891053 |
| SimpleNonHalter | 4 | 4 | 0.54891053 |
| BB5Champion | 3 | 2 | 0.54891053 |
| CurrentChampion | 4 | 4 | 0.54891053 |
| Antihydra | 3 | 2 | 0.54891053 |

**观察**：
- 格概念数从 2 提升到 3-4（略有改善）
- 不同 TM 的格结构略有不同（3 vs 4 概念）
- **但 ρ_top 对所有 TM 完全相同：0.54891053**

## §4. 根因分析

### 4.1 问题 1：FCA 格太小

**现象**：即使有 25,000 个对象，格也只有 3-4 个概念。

**原因**：FCA 概念数 = 不同属性组合的数量。v2 构造有 26 个属性（4 分箱位置 + 8 全局 + 3 状态组 + 11 转移），
理论上最多 2^26 个概念，但实际中：

- 分箱位置属性（4 个）创建了 4 种组合
- 全局属性（8 个）在所有对象间高度共享
- 状态组属性（3 个）仅按状态分组
- 转移属性（11 个）仅按状态-符号对分组

实际不同的属性组合远少于对象数，导致 FCA 格退化为少数几个概念。

**数学描述**：设对象集 O = {o₁, …, o_N}，每个对象 o_i 的属性向量为 a_i ∈ {0,1}^m。
FCA 概念数 = |{a_i↑ : i = 1, …, N}|，即不同属性向量的"上闭包"数。
当属性向量的多样性低时，概念数也低。

### 4.2 问题 2：N 算子对小格不敏感

**诊断实验**：对不同大小的链格（2-100 个概念）计算 ρ_top。

预期结果：测试 N 算子是否对不同格大小产生不同 ρ_top。

**假设**：若 ρ_top 对格大小敏感，则问题在于格太小；若 ρ_top 始终为 0.5489，
则问题在于 N 算子对小格退化为固定点。

### 4.3 问题 3：形式背景构造的深层矛盾

**理论矛盾**：引理 3.2 声称格同构于 TM 的状态转移子图，但：

- 状态转移子图是**有向图**（节点 = 配置，边 = 转移）
- FCA 概念格是**完备格**（每个子集有上确界和下确界）

两者的结构根本不同：
- 有向图的传递闭包是**偏序**，但不一定是格（可能缺少 meet 或 join）
- FCA 格总是完备格

**关键洞察**：FCA 格不是状态转移图的"编码"，而是形式背景的"概念结构"。
状态转移图的传递闭包和 FCA 格不一定同构，除非形式背景被精心构造。

### 4.4 问题 4：ρ(J) = 0.5489 的起源

**猜想**：ρ(J) = 0.54891053 是 N 算子在"最小格"（2-4 个概念）上的默认不动点。

对于极小的格，5 维动力学退化为低维，Jacobi 矩阵的谱半径由格拓扑（而非具体权重）决定。
链格（2-4 个概念）的拓扑基本相同（一条链），因此 ρ(J) 相同。

**验证方法**：测试不同格拓扑（链、菱形、布尔格）的 ρ(J) 是否不同。

## §5. 可能的修正方案

### 5.1 方案 A：增大格规模

**思路**：增加属性维度，使不同的对象产生不同的属性向量，从而增加概念数。

**具体方法**：
- 使用更多分箱位置属性（不只 mod 2，而是 mod 4、mod 8 等）
- 添加更多全局属性（不同窗口大小下的密度、熵等）
- 使用原始位置-符号属性（不合并），但将对象限制为实际可达配置

**风险**：属性维度增加后，概念数爆炸 → 计算不可行。

### 5.2 方案 B：直接使用配置图作为格

**思路**：跳过 FCA 映射，直接将 TM 的配置图（或其传递闭包）作为格，应用 N 算子。

**挑战**：
- 配置图不一定是格（可能缺少 meet/join）
- 需要将配置图补全为格（Dedekind-MacNeille 完备化）
- 配置图的 Dedekind-MacNeille 完备化可能非常大

### 5.3 方案 C：Collatz 级抽象

**思路**：对于 Antihydra，利用已知的 Collatz 级行为（A(a, b) 参数化），
直接构建 (a, b) 参数空间的格。

**具体方法**：
- 模拟 Antihydra 至 N 步，每步记录 (a, b) 值
- 对象 = 时间步，属性 = a mod k, b mod k, a/b 比值等
- FCA 格捕获 (a, b) 序列的"概念结构"

**优势**：格规模可控（属性数 = 维数），且 (a, b) 序列具有丰富的结构。
**劣势**：仅适用于 Antihydra，不适用于一般 TM。

### 5.4 方案 D：多步可达到性格

**思路**：使用 TM 的转移规则构建"k 步可达到性"矩阵，作为形式背景。

**定义**：对每个深度 d = 1, …, D，定义属性 "can_reach_q_in_d_steps"。
对象 = (state, symbol) 对。关联：若从 (q, γ) 出发可在 d 步内到达 q'，则关联。

**优势**：格规模由 D 控制，且不同 TM 的可达到性模式不同。
**劣势**：仅使用 TM 的转移规则，不使用实际计算轨迹。

### 5.5 方案 E：部分配置的 FCA 格

**思路**：不只使用单个配置，而是使用"配置对"（前一步 + 后一步）作为对象。

**定义**：对象 = (ID_{M,L}(t), ID_{M,L}(t+1))，即一步转移。
属性 = 前状态、后状态、读写符号、移动方向、带变化等。

**优势**：对象数 = 模拟步数，属性 = 转移的所有可能特征。
**劣势**：格可能仍然太小。

## §6. 当前诊断实验设计

### 6.1 ρ(J) 对格大小的敏感性

**实验**：对链格（n=2, 3, 4, 5, 10, 20, 50, 100）和菱形格计算 ρ_top。

**预期**：
- 若 ρ_top 随 n 变化 → 格太小是根因，增大格可解决

**实验**：对同一 TM，变化属性维度（m = 4, 8, 16, 32, 64），观察概念数变化。

**预期**：概念数应随属性维度增加而增加，但可能饱和。

**实验**：对 Antihydra，比较三种构造的格大小和 ρ_top：
- v1：静态构造（所有三元组）
- v2：轨迹构造（当前方法）

## §7. 理论结论（暂定）

### 7.1 当前状态

v1 和 v2 的构造均未能产生足够丰富的 FCA 格，导致 N 算子无法区分不同 TM。

**核心问题**：TM 的计算轨迹在 FCA 形式背景中产生的是"平坦"的对象集（每个配置是唯一的），
而非"层次化"的对象集（概念之间应该有丰富的偏序关系）。

### 7.2 理论意义

ISD 框架的 N 算子要求格具有丰富的概念层次结构才能产生有意义的谱半径。
TM 的配置图本质上是"链"（或"树"），其 FCA 概念格退化为少数几个概念。

**关键洞察**：FCA 格捕获的是"概念层次"，而 TM 的计算轨迹是"时间序列"。
这两个结构在数学上是不兼容的——除非我们找到一种方式将时间序列嵌入概念层次。

### 7.3 前进方向

1. **完成诊断实验**（§6），确认 ρ(J) 对小格的敏感性
2. **若 ρ(J) 对小格不敏感**：需要根本不同的方法（方案 B 或 D）
3. **若 ρ(J) 对大格敏感**：需要找到增大格规模的方法（方案 A 或 C）
4. **若方案 C 对 Antihydra 有效**：可以宣称 ISD 框架对类 Collatz 机器有效，但不等价于一般性判定
