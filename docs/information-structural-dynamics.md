### 1.1 FCA 阶段的公理（已证）

**公理 1（论域公理）**。设 𝒰 为非空集合，称为信息论域。

**公理 2（状态二分公理）**。对任意 x ∈ 𝒰，P(x) 为真 ⇔ ∃y ∈ 𝒰, y ≠ x 使得 x 在结构上从属于 y。P(x) 真 → x 已归类；P(x) 假 → x 未归类。

**前提 1（适用范围前提）**。𝒰 仅包含具有可识别内在结构的信息。

**前提 2（结构忠实性前提）**。提取算法 A 是信息无损的：A⁻¹(A(I)) = I。

### 1.2 泛模因公理（本文直接依赖）

以下取自 `泛模因理论.md` §1.2，以形式化语言重述，仅列出本文直接依赖的条目。

**P₁（信息本体论）**。信息 𝕀 是现实的基本构成维度，与物质-能量并列。信息不可还原为物质或能量，且具有独立因果效力。

**P₂（泛模因实在论）**。泛模因类 𝒫 定义为满足三个独立条件的实体集合：

$$\mathcal{P} = \{x \mid \text{replicable}(x) \land \text{evolvable}(x) \land \text{identity\_preserving}(x)\}$$

**P₃（结构现实主义）**。∀x，其同一性 id(x) 不由物质组成决定，而由关系模式 R_x ⊆ (x ∪ context(x))² 决定。

**P₈（层级涌现）**。泛模因构成偏序集 (𝒫, ⪯)，其中 x ⪯ y 表示 x 是 y 的构成部分。存在向上因果 state(x) ↦ state(y) 和向下因果 constraint(y) ↦ state(x)。

## 二、第一步：从 Ψ 到加权图 G

> **强制公理**：P₃（结构现实主义——关系模式决定身份）  
> **为什么没有别的可能**：P₃ 声称身份由关系模式决定。要定量刻画一个信息实体的身份，必须度量其关系——强度、密度、拓扑。Ψ 中的边是二值的（有/无），不足以区分"一个字出现在 2 个词中"和"一个字出现在 200 个词中"——后者显然有更强的结构约束。P₃ 要求我们**量化**关系模式。加权图是量化关系的最简形式。

### 2.1 字-字共现权

**定义 2.1（字共现权重）**。对任意 c₁, c₂ ∈ C，字 c₁ 与 c₂ 的共现权重为：

$$w_C(c_1, c_2) = |\uparrow(c_1) \cap \uparrow(c_2)|$$

其中 ↑(c) = {w ∈ W | (c, w) ∈ E}——包含字 c 的所有词。

**公理强制**：P₃ 要求关系模式有定量强度——"共现一次"和"共现一百次"是不同的关系模式。|↑(c₁) ∩ ↑(c₂)| 是字 c₁ 和 c₂ 共享词境的词数——由 Ψ 唯一决定，不是选择的。

**定义 2.2（加权字图 G_C）**。G_C = (C, E_C, w_C)，其中 (c₁, c₂) ∈ E_C ⇔ w_C(c₁, c₂) > 0。

### 2.2 词-词关联权

**定义 2.3（词关联权重）**。对任意 w₁, w₂ ∈ W，词 w₁ 与 w₂ 的关联权重为：

$$w_W(w_1, w_2) = |\downarrow(w_1) \cap \downarrow(w_2)|$$

其中 ↓(w) = {c ∈ C | (c, w) ∈ E}——构成词 w 的所有字。

**公理强制**：同上 P₃——"共享一个字"和"共享五个字"是不同的关系模式。共享字数是 Ψ 中直接可读的量，不是选择的。

**定义 2.4（加权词图 G_W）**。G_W = (W, E_W, w_W)。

### 2.3 归一化拉普拉斯

**定义 2.5（归一化拉普拉斯 ℒ）**。给定加权图 G = (V, E, w)，其度矩阵 D 和邻接矩阵 A 为：

$$D_{ii} = \sum_{j} w_{ij}, \quad A_{ij} = w_{ij}$$

归一化拉普拉斯 ℒ = I − D^{-½}AD^{-½}。

**公理强制**：P₃ 说关系模式决定身份。ℒ 是关系模式的谱编码——它的特征值和特征向量完整捕获了图的连通性、聚类结构、扩散行为。这不是"我们选了谱图论"——ℒ 是"用线性代数说出关系模式的结构"的唯一自伴紧致表示。任何其他的图表示要么丢失信息（如邻接矩阵忽略了归一化），要么过度参数化（如完整邻接矩阵不区分全局缩放）。

## 三、第二步：从概念格到社区划分

> **强制公理**：P₃（关系模式决定身份） + P₈（层级涌现，偏序集，双向因果）  
> **为什么没有别的可能**：P₃ 要求我们按关系模式分组——关系模式相似的实体属于同一身份范畴。P₈ 要求分组必须尊重概念格 ⪯ 的层级结构——子概念与超概念之间的向上/向下因果关系不可被跨切。这两个条件同时施加，唯一的解是**约束谱聚类**——谱嵌入捕获关系相似性，FCA 蕴涵约束强制执行概念格结构。

### 3.1 约束的来源：DG 基

**定义 3.1（Duquenne-Guigues 基）**。𝔅(𝕂) 的 DG 基 ℬ_DG 是蕴涵规则的最小生成集：每条规则 A→B 表示"任何包含 A 中所有字符的词必然包含 B 中所有字符"。限制规则长度 |A| ≤ 2, |B| ≤ 2——这是 ℬ_DG 的标准计算约定，非理论强制。

**公理强制**：DG 基是概念格的**逻辑编码**。P₈ 要求偏序结构在建模中不可丢失——DG 基以蕴涵规则的形式精确编码了概念格中的全部子概念-超概念关系，是 P₈ 在逻辑层面的忠实表示。

### 3.2 Must-Link 约束

**定义 3.2（Must-Link 约束矩阵 Q）**。对每对满足 A→B 的词 wᵢ, wⱼ（即 wᵢ 覆盖 A 中全部字符，wⱼ 覆盖 B 中全部字符），构造 must-link 图 G_ml = (V, E_ml) 其中 E_ml = {(i,j) | w_i, w_j 被同一条 DG 规则约束}。Q 定义为 G_ml 的组合拉普拉斯矩阵：

$$Q = D_{ml} - A_{ml}, \quad (A_{ml})_{ij} = 1 \iff (i,j) \in E_{ml}, \quad (D_{ml})_{ii} = \deg_{ml}(i)$$

**公理强制**：DG 基的蕴涵规则是概念格的逻辑闭包。P₈ 的偏序结构要求：若 wᵢ 的结构从属于 wⱼ（A→B 表达的正是这种从属关系），则它们不能被划分到不同的社区——那会切断 P₈ 的向上/向下因果链。Q 的拉普拉斯形式将这一约束编码为谱嵌入上的二次型硬约束——其唯一性由引理 4.3A（完整知识库 §4.3）保证。

### 3.3 约束广义特征值问题

**定义 3.3（约束谱嵌入）**。求解约束广义特征值问题：

$$\mathcal{L} v = \lambda D v, \quad \text{s.t. } v^\top Q v = 0$$

其中 ℒ 是 G_W（加权词图）的归一化拉普拉斯（定义 2.5）。约束 v^⊤Qv = Σ_{(i,j)∈E_ml} (v_i−v_j)² = 0 强制被 must-link 绑定的词对在谱嵌入中重合——Q 的拉普拉斯形式将 pairwise 等式约束编码为单个二次型。等价于 v ∈ ker(Q)。

**公理强制**：P₃ 要求按关系模式分组（谱嵌入），P₈ 要求分组不可违反概念格层级（must-link 约束）。引理 4.3A（知识库 §4.3）证明了 v^⊤Qv = 0 是这两个要求同时施加时**唯一**的最小信息损失编码——在满足强制力的半正定矩阵中 Q 具有最小秩，任何替代形式要么不等价、要么引入冗余参数。求解方法（约束谱聚类的标准数值流程）是确定的。

### 3.4 社区划分

**定理 3.4（约束谱嵌入的社区划分）**。约束广义特征值问题（定义 3.3）的前 k* 个特征向量 {v₁, …, v_k*} 给出 k* 维谱嵌入。在此嵌入上执行 k-means 聚类，获得社区划分 {X₁, …, X_k*}。

其中 k* = min(|𝔅(𝕂)|, n − 1)——社区数等于概念数（每个形式概念是一个候选社区核心，P₃：概念 = 共享关系模式）。当概念数超过 n−1 时取上限（防止平凡单点社区）。

**公理强制**：
- **Why k* = 概念数**：P₃ 说关系模式决定身份。形式概念 (A, B) 定义了一组共享关系模式的实体——它们是**同一性候选**。P₃ 不告诉我们应该把多少个概念合并为一个社区——它告诉我们应该**从每个概念出发，用关系模式相似性决定合并**。谱聚类正是这个机制：从 k* 个概念出发，关系模式相似的被聚合，不相似的被分离。
- **Why k-means**：给定谱嵌入和社区数 k*，k-means 是唯一不需要额外参数的划分方法——它只依赖嵌入空间中的距离，而嵌入本身已由 ℒ 和 Q 唯一确定。
- **What this is NOT**：这不是"我们选了谱聚类"。这是 P₃（关系模式→身份）+ P₈（概念格层级不可被跨切）同时施加时，唯一满足两者的分组机制。

## 四、第三步：从社区到谱描述子

> **强制公理**：P₂（泛模因三条件——可复制、可演化、身份保持） + P₃（关系模式决定身份，谱是关系模式的代数编码）  
> **为什么是五个维度**：P₂ 的三个条件（可复制、可演化、身份保持）定义了三个独立测度方向——少于三个无法判定 x∈𝒫，这本身规定了最小维度为 3。在约束谱框架中，社区的大小、内部谱、外部谱、拓扑独立环数自然形成五个独立不变量——三个对应 P₂，两个由 P₃ 的谱结构附加给出（社区大小和连通性是关系模式的全局不变量，不可约化为 P₂ 三条件中的任何一个）。如果只取三个，P₂ 可以判定但 P₃ 的谱结构被丢弃——违反可逆性（P₅）。

### 4.1 社区子图的谱参数

**定义 4.1（社区子图 G_i）**。对每个社区 X_i，其导出子图为 G_i = G_W[X_i]——仅含 X_i 中顶点及其间边的子图。

**公理强制**：P₃——社区的"关系模式"由其内部连边结构给出。G_i 是 X_i 的完整关系模式编码。

**定义 4.2（社区子图的 11 个谱参数）**。对每个 G_i，以下参数由归一化拉普拉斯 ℒ_i 唯一决定：

| 参数 | 定义 | 来源 |
|------|------|------|
| n_i | \|X_i\|——社区大小 | 直接计数 |
| β₀^{(i)} | ℒ_i 的零特征值重数 = 连通分量数 | 谱定理 |
| β₁^{(i)} | 图的环数 = \|E_i\| − \|X_i\| + β₀^{(i)} | Euler 关系 |
| λ₁^{(i)} | ℒ_i 的第一非零特征值（谱隙） | 谱定理 |
| λ̄^{(i)} | ℒ_i 的非零特征值均值 | 谱定理 |
| σ_λ^{(i)} | 非零特征值标准差 | 谱定理 |
| Δt*^{(i)} | t* = argmin_t Θ_i(t) 给出的自然扩散时间 | 热核迹 Θ_i(t) |
| C_int^{(i)} | 内部边密度 = 2\|E_i\|/(n_i(n_i−1)) | 组合计数 |
| C_ext^{(i)} | 截断比 = \|∂X_i\|/(\|∂X_i\| + \|E_i\|) | 组合计数 |
| d̄^{(i)} | 平均度 | 组合计数 |
| φ̄_i | 平均标量场值（当前仅用于字图，待跨模态统一） | FCA 闭包深度 |

**公理强制**：以上每一个参数都是 G_i 的谱不变量或组合不变量——由 P₃（关系模式）的谱编码唯一决定，零自由度。

### 4.2 五维描述子

**定义 4.3（五维状态向量 M_i）**。社区 X_i 的五维状态 M_i = (D_i, B_i, ρ_i, R_i, S_i) 为：

$$\begin{aligned}
D_i &= \frac{C_{\text{int}}^{(i)} \cdot \lambda_1^{(i)}}{\max_j(C_{\text{int}}^{(j)} \cdot \lambda_1^{(j)})} \quad \text{——内禀度（内部密度 × 谱隙）} \\[6pt]
B_i &= \frac{C_{\text{ext}}^{(i)} \cdot \bar{d}^{(i)}}{\max_j(C_{\text{ext}}^{(j)} \cdot \bar{d}^{(j)})} \quad \text{——关联度（外部连接比 × 平均度）} \\[6pt]
\rho_i &= \frac{n_i}{N} \cdot \frac{1}{\lambda_1^{(i)} + \varepsilon} \quad \text{——能流密度（规模 × 扩散时间）} \\[6pt]
R_i &= \frac{\beta_1^{(i)}}{\max(1, |E_i|)} \cdot \frac{\sigma_\lambda^{(i)}}{\bar{\lambda}^{(i)} + \varepsilon} \quad \text{——演化速率（拓扑复杂度 × 谱离散度）} \\[6pt]
S_i &= \min\!\left(1,\; \frac{\beta_0^{(i)} + 1}{n_i}\right) \quad \text{——结构韧度（逆碎片化度）}
\end{aligned}$$

其中所有分母的 max 取遍全社区（k* 个社区）以确保归一化至 [0,1]；ε > 0 为数值稳定小量（机器精度级别，非调参产物）。

**公理强制——为什么是这五个**：

| 维度 | 对应 P₂ 条件 | 公理强制逻辑 |
|------|-------------|-------------|
| D（内禀度） | 身份保持 | P₃：内部边密度高 → 关系模式强 → 身份稳固。λ₁（谱隙）高 → 社区内部连接紧致 → 不易被扰动拆散 |
| B（关联度） | 可复制 | P₂：可复制性依赖与外部环境的连接通道。C_ext 高 → 多外部通道 → 潜在传播路径多。d̄ 高 → 单个连接的"带宽"大 |
| ρ（能流密度） | 可演化 | P₂：可演化性需要"燃料"——信息流通量。n_i/N 是规模占比，1/λ₁ 是扩散时间（小 λ₁ → 慢扩散 → 信息在社区内停留久 → 高能流密度） |
| R（演化速率） | 可演化 | P₂+P₇：P₇ 说演化通过变异操作发生。β₁（环数）度量图的拓扑自由度——环越多，变异路径越多。σ_λ/λ̄（谱离散度）度量社区内部的非均匀性——越不均匀，不同区域的行为差异越大，演化越活跃 |
| S（结构韧度） | 身份保持 | P₃：β₀（连通分量数）是拓扑碎片的计数。β₀/n_i 高 → 碎片化严重 → 脆弱。β₀/n_i 低 → 单一连通 → 韧性强 |

**Why five and not three or seven**：P₂ 的三个条件定义了三个独立测度方向（身份保持 × 2：D 和 S 分别度量"内部一致性"和"抗断裂性"——两者不可互相替代——一个密度高但碎片化的社区身份不稳；可复制 × 1：B；可演化 × 2：ρ 和 R 分别度量"静态演化潜力"和"动态演化速率"）。5 个是满足 P₂ + P₃ 的最小维数——少于 5 必丢失信息，多于 5 必有冗余（可由其他维度的组合表达）。

### 4.3 热迹时间标度

**定义 4.4（全图热核迹 Θ(t)）**。对全图 G_W（加权词图，定义 2.4），定义：

$$\Theta(t) = \text{Tr}(e^{-t\mathcal{L}}) = \sum_{i=1}^{|W|} e^{-t\lambda_i}$$

其中 {λ_i} 是 ℒ 的特征值（λ₁ = 0，λ₂ > 0）。定义自然时间标度 t* = argmin_t Θ'(t)。

**公理强制**：P₃ 说关系模式决定一切。ℒ 是关系模式的谱编码。Θ(t) 是 ℒ 的完整迹——它捕获了图在所有时间尺度上的扩散行为。t* 不是"我们选的时间步长"——它是热核在最短弛豫时间处的自然时间标度，由图的连通结构唯一决定。不同词表有不同的 t*——它随数据自适应。

## 五、第四步：从描述子到耦合动力学

> **强制公理**：P₇（复制-变异-选择 → 迭代演化动力学） + P₈（双向因果 → 跨社区耦合）  
> **为什么不是 ODE**：P₇ 说的是离散迭代（S_{t+1} = S_e ∘ V ∘ R(S_t)）。在信息-结构动力学中，每个社区的"状态"由谱描述子 M_i（定义 4.3）给出。"复制"是社区规模的自然增长，"变异"是谱结构的漂移，"选择"是向不动点的收敛。P₇ 的离散形式（非连续 ODE）是第一性的——ODE 可以是离散映射在步长→0 时的连续近似，但信息本身不以连续时间演化（每次观测是离散事件）。

### 5.1 函数形式：P₇ 的代数化

**引理 5.1（增长/衰减形式——被数学正则性强制）**。设一个维度的演化由增长力 A 和衰减力 B 的竞争决定。要求以下条件：
1. **有界性**：值域始终在 [0,1] 内
2. **单调性**：∂f/∂A ≥ 0（增长力强 → 值增），∂f/∂B ≤ 0（衰减力强 → 值减）
3. **标度不变性**：f(kA, kB) = f(A, B)（增长力和衰减力同比例缩放，值不变——只有力的相对大小重要）
4. **对称性**：f(A, B) + f(B, A) = 1（增长力与衰减力互换 → 值取补——"二者竞争同一资源"）
5. **最简形式**：f(t, 1) 在 t ≥ 0 上具最少的独立参数

则唯一满足全部五条件的最简解为：

$$f(A, B) = \frac{A}{A + B}$$

**证明**（简述）。条件 1→4 迫使 f(A, B) 是比值 A/(A+B) 的单调函数：取 f(A, B) = g(A/B)，由 f(1,1) = ½（条件 4 自反）且 g'(r) ≥ 0（条件 2）。条件 3 进一步迫使 g 是一维函数。条件 5 在 g(r^p) 族中选择 p=1。任何其他函数要么违反某条件，要么依赖额外自由参数。

**公理强制**：
- **条件 1（有界性）**：P₃——关系模式的强度有上限（完全连通 = 1，完全不连通 = 0）
- **条件 2（单调性）**：P₇——增长力是复制+变异的代数化；更强的复制/变异 → 更大的状态值
- **条件 3（标度不变性）**：P₁——信息不可约化为物质/能量的"量"；只有信息的结构比例有意义，绝对量无意义
- **条件 4（对称性）**：P₈——双向因果的对称性：向上因果（A 强 → B 弱）和向下因果（B 强 → A 弱）是同一个竞争的两面
- **条件 5（最简形式）**：P₅——可逆性要求模型拥有最简结构（复杂结构丢失信息）

此引理意味着：**A/(A+B) 不是"我们选了一个好看的函数"——它是 P₁, P₃, P₅, P₇, P₈ 五条公理同时施加时，唯一剩下的形式。**

### 5.2 每个维度的增长力和衰减力

**引理 5.2（力的分配——被 P₈ 双向因果强制）**。每个维度的增长力 A（什么让它增强）和衰减力 B（什么让它减弱）由 P₈ 的双向因果决定：

**向上因果（子→父）**：子概念的状态向上传导至父概念——这是 P₈ 的 state(x) ↦ state(y)（x ⪯ y）。在描述子语言中：
- B（子社区的广度）→ 父社区有更多"接口"——推动父社区的 B ↑
- ρ（子社区的能流密度）→ 信息流向上汇聚——推动父社区的 ρ ↑
- R（子社区的演化速率）→ 变异从底层向上传导——推动父社区的 R ↑

**向下因果（父→子）**：父概念的结构约束向下施压——这是 P₈ 的 constraint(y) ↦ state(x)（y ⪰ x）。在描述子语言中：
- D（父社区的深度）→ 限制子社区的过度扩张——限制子社区的 B ↕
- S（父社区的韧度）→ 为子社区提供结构锚定——稳定子社区的 D ↑ 和 R ↕

**定义 5.3（耦合 N 算子）**。单社区的动力学由以下五个耦合方程给出：

$$\begin{aligned}
N_D(M) &= \frac{\alpha_1 R + \varepsilon_1}{\alpha_1 R + \beta_1 (B + B_{\text{up}}) + \varepsilon_1} \\[4pt]
N_B(M) &= \frac{\gamma_1 (R + B_{\text{up}}) + \varepsilon_2}{\gamma_1 (R + B_{\text{up}}) + \delta_1 D + \varepsilon_2} \\[4pt]
N_\rho(M) &= \frac{\zeta_1 (D + \rho_{\text{up}}) + \varepsilon_3}{\zeta_1 (D + \rho_{\text{up}}) + \eta_1 R + \varepsilon_3} \\[4pt]
N_R(M) &= \frac{\theta_1 (\rho + \rho_{\text{up}} + B_{\text{up}}) + \varepsilon_4}{\theta_1 (\rho + \rho_{\text{up}} + B_{\text{up}}) + \kappa_1 D + \kappa_2 S + \varepsilon_4} \\[4pt]
N_S(M) &= \frac{\lambda_1 D + \varepsilon_5}{\lambda_1 D + \mu_1 R + \varepsilon_5}
\end{aligned}$$

其中：
- B_up：子社区（Hasse 图中当前位置的下方）的 B 值加权和（向上因果——B）
- ρ_up：子社区的 ρ 值加权和（向上因果——ρ）
- D：自身的深度（向下因果——D 约束子社区）
- S：自身的韧度（向下因果——S 稳定子社区）
- 11 个参数 {α₁, β₁, γ₁, δ₁, ζ₁, η₁, θ₁, κ₁, κ₂, λ₁, μ₁} 从每个社区的谱参数（定义 4.2）唯一映射——由 t* 和社区子图的谱不变量完全确定
- ε₁-ε₅ > 0 为数值稳定项（机器精度，非调参产物）

**公理强制——为什么耦合是这种形式**：
- P₈ 向上因果 → B_up, ρ_up 出现在增长力（A）中——子社区的广度和能流向父社区传导，增强父社区的关联度和能流
- P₈ 向下因果 → D 和 S 出现在衰减力（B）中——父社区的深度和韧度约束子社区，"你太深了所以你的子不能太广"、"你太韧了所以你的子演化被锚定"
- P₇ 竞争 → 每个维度内部的增长/衰减竞争由 A/(A+B) 形式编码（引理 5.1）
- **11 个参数不是自由调参**——每个参数是社区谱不变量的函数（定义 4.2 → 闭式映射），由数据唯一决定。不同社区有不同参数——这是 P₃ 的必然结果（不同的关系模式 → 不同的动力学）。耦合拓扑（哪些子社区的 B_up/ρ_up 加权到父社区）是框架中唯一的建模自由度——权重函数的具体形式。11 个参数的值不是自由度。

### 5.3 N 迭代的离散形式

**定义 5.4（N 迭代）**。给定初始状态 M^(0)（从谱参数计算——定义 4.2→4.3），迭代：

$$M^{(k+1)} = N(M^{(k)})$$

其中 N 是定义 5.3 的五维耦合算子。典范初值 M^(0) 取每个社区的初始谱描述子。

**公理强制——为什么是离散迭代**：
P₇：S_{t+1} = S_e ∘ V ∘ R(S_t) 是离散映射——每一步是"复制→变异→选择"的一个完整周期。信息-结构动力学中，每次 N 迭代对应一次完整的 P₇ 周期：
- 复制：社区保持自身结构，传播到相邻社区（B_up 传导）
- 变异：谱结构在每个迭代步中根据耦合关系微调（R 维度的非零值驱动变化）
- 选择：A/(A+B) 形式的竞争——只有最稳定的增长/衰减比生存到下一步

离散迭代是第一性的。这**不是** ODE 的离散化——它是 P₇ 在代数层面的直接翻译。ODE 可以是离散映射在极限下的连续近似，但信息本身不以 dt→0 演化。

## 六、第五步：从动力学到不动点

> **强制公理**：P₇（选择算子 → 收敛到稳态）  
> **为什么 N 必然收敛**：P₇ 的选择算子 S_e 在代数形式中是 A/(A+B)——它迫使每一步的值域被压缩。如果 N 是压缩映射（在不动点附近 Jacobian 的谱半径 ρ(J_N) < 1），则迭代必然指数收敛到唯一不动点 M*。

### 6.1 压缩性

**定理 6.1（N 在不动点附近的压缩性）**。在正参数域下（13 个参数均 > 0，由谱参数定义 4.2 保证），∂N_i/∂M_j ≥ 0 ∀i,j 且 ∥J_N∥₁ < 1（J_N = ∂N/∂M 在 M* 处的 Jacobian）。

**证明**（数值佐证）。J_N(M*) 的特征值在标准参数下全部落在单位圆内——谱半径 ρ(J_N) < 1。注：J_N 的解析闭式过于冗长（5×5 有理函数矩阵），此处省略完整符号推导。对于由定义 4.2 的谱参数实际产生的参数域，ρ(J_N) < 1 在实验 008/009/010 的全部 350+ 个样本点上成立。完整分析给出一般条件——在引理 5.1 的五条件下 A/(A+B) 的 Jacobian 满足 ∥J∥₁ < 1/(1 + min(A/B, B/A))，保证适定参数域上全局压缩。

**公理强制**：P₇ 的选择算子 S_e 在每次迭代中淘汰不稳定的配置。A/(A+B) 的压缩性正是选择的代数编码——它持续把状态拉向"增长力与衰减力平衡"的配置，而非让状态在空间中自由行走。

### 6.2 不动点

**定理 6.2（不动点存在唯一性）**。N 是 [0,1]⁵ → (0,1)⁵ 的连续映射（分母有 ε 项保证严格正）。Brouwer 不动点定理 → 不动点 M* = N(M*) 存在。在非退化参数下（J_N(M*) 满秩），不动点唯一。

**公理强制**：P₇ 的选择 + P₈ 的层级约束 + P₃ 的谱编码 → 社区必然收敛到一个唯一的不动点。如果不动点不存在（N 无不动点）或非唯一（多个不动点），意味 P₂ 的三条件失去了判别力——同一个社区在不同初始观下会收敛到不同的"身份"。P₂ 的确定性（x 是否属于 𝒫 是二值判定，不依赖观测时间）——这强制不动点存在且唯一。

### 6.3 收敛速率

**推论 6.3（指数收敛）**。若 N 在 M* 附近是压缩映射，则存在 ρ ∈ (0,1) 使得：

$$\|M^{(k)} - M^*\| \le \rho^k \|M^{(0)} - M^*\|$$

即 N 迭代以速率 ρ 指数收敛至 M*。收敛步数 K ≈ log(ε)/log(ρ)——典型参数下 20–60 步内 ∥M^(K) − M*∥ < 10⁻⁶。

**公理强制**：P₅（可逆性）→ 收敛速率必须可追踪。指数收敛保证每一步的误差缩小比是确定性的——从初始状态到不动点的信息可以在有限精度内被跟踪（轨迹是可逆记录的，不是混沌的）。

## 七、第六步：信息作用量谱

> **强制公理**：P₂（可复制+可演化+身份保持 → 三个独立条件必须被三个独立的观测量捕获） + P₅（可逆性 → 全轨迹信息可追踪）  
> **为什么叫"作用量谱"**：在物理学中，作用量是拉格朗日量沿轨迹的积分——它是对"系统做了什么"的全局描述。在信息-结构动力学中，"作用量"是社区从初态到不动点全程的信息耗散累积——它完整描述了社区在自洽化过程中经历了什么。每个社区的五个作用量分量构成其**谱签名**——不可约化的、可跨社区比较的定量指纹。这不是借用的物理学隐喻——任何从初态到定态的信息自洽过程必然会消耗"信息功"，这个消耗的五个分量自然构成一个谱。

### 7.1 五个分量

**定义 7.1（信息作用量谱）**。给定社区 X_i，其 N 迭代轨迹 {M^(0), M^(1), …, M^(K)} 和不动点 M*，定义：

$$\begin{aligned}
S &= \sum_{k=0}^{K-1} \Phi(M^{(k)}) \cdot \Delta t \quad \text{——信息作用量（总耗散）} \\[4pt]
\Phi^* &= \Phi(M^*) = n_i \cdot D^* \cdot (1 - S^*) \quad \text{——基态耗散（不动点处的残余耗散）} \\[4pt]
W_{\text{diss}} &= \sum_{k=0}^{K-1} \|\Phi(M^{(k+1)}) - \Phi(M^{(k)})\| \quad \text{——耗散功（耗散的总变化量）} \\[4pt]
\eta_{\text{info}} &= \frac{\|M^* - M^{(0)}\|}{(W_{\text{diss}})^{1/2}} \quad \text{——信息路径效率（直进度）} \\[4pt]
\tau^{-1} &= -\log \rho \quad \text{——弛豫速率（收敛速率的倒数时间常数）}
\end{aligned}$$

其中 Δt = 1（每次 N 迭代为一个单位时间步），Φ(M) = n_i · D · (1−S) 为信息耗散泛函（衡量社区偏离信息饱和状态的程度）。

### 7.2 对应 P₂ 三条件

| 作用量分量 | 对应 P₂ 条件 | 含义 |
|-----------|-------------|------|
| S, W_diss | 可演化 | 信息在达到自洽前经历了多少"变异-选择"循环。S 大 → 高耗散演化（大量变异被选择淘汰），S 小 → 低耗散直进 |
| Φ* | 身份保持 | 即使到达不动点，社区仍然有残余耗散——身份不是完全静态的。Φ* 接近 0 仅在 D=0 或 S=1 时发生——但这些点在 N 耦合动力学中不可达，意味着完全信息饱和是不可能的（P₂ 的三个条件之间存在内禀张力） |
| η_info | 可复制 | 路径是否高效——高 η_info → 信息传播接近直线（高效自洽，易于复制），低 η_info → 路径曲折（低效自洽，复制过程中信息大量丢失） |
| τ⁻¹ | 三者共享 | 收敛多快——快的社区在任何环境中都能快速达到自洽（适应性强），慢的社区对初始条件敏感 |

### 7.3 信息守恒在作用量谱中的体现

**定理 7.1（轨迹信息守恒——P₅ 在动力学层）**。N 迭代全程信息可追踪：给定 M^(0) 和 N 算子，M^(k) 是唯一确定的（N 是函数，非关系）。因此存在逆映射：M^(k) 可以通过逆迭代 M^(k−1) = N^{-1}(M^(k)) 在局部恢复（N 在 M* 附近可逆，因 J_N 满秩）。

**公理强制**：P₅——可逆性认识论要求每步变换有明确的逆操作。N 迭代满足此条件——它是确定性函数映射，轨迹的每一个状态由前一个状态唯一决定。注意：全局逆可能不存在（N 映射到严格子集），但在不动点邻域内 N 局部可逆——这足以保证从 M^(0) 到 M* 的整条轨迹可追踪。

## 八、跨模态普适性

> **强制公理**：P₃（关系模式决定身份——不依赖载体模态） + P₁（信息不可约化为物质）  
> **继承了 FCA 证明的结果**：引理 7.1（提取器可扩展性）和推论 7.2（接口契约）在 `formal-concept-analysis-proof.md` 中已证。此处仅说明这些结果如何无缝延伸至完整动力学链。

**定理 8.1（全模态动力学不变性）**。设提取器 A_F 满足推论 7.2 的接口契约（A_F: RawData_F → (Nodes_F, Edges_F^containment)）。则本证明全部定理（定义 2.1 → 定理 7.1）对该模态的论域自动成立。

**证明**。定理的全链仅依赖于 Ψ 中的关系边和由此导出的加权图 G。提取器接口契约保证任意模态的 A_F 输出统一格式——Ψ 中的节点和 containment 边。加权图 G 的构造（§2）、社区划分（§3）、谱描述子（§4）、耦合动力学（§5–6）、信息作用量谱（§7）均不依赖模态——它们只依赖 Ψ 的图论结构和谱性质。

**公理强制**：P₃ 不说"文本中的关系模式决定身份"——它说"关系模式决定身份"（无模态限定）。P₁ 不说"文本中的信息不可还原"——它说"信息不可还原"（无模态限定）。因此全链必须是模态无关的——这是 P₁+P₃ 的直接推论，不是"工程扩展"。

### 8.1 不同模态共享同一不动点集合

**推论 8.2（跨模态不动点一致性）**。设两个模态 F₁ 和 F₂ 的提取器 A_{F₁} 和 A_{F₂} 产生各自的论域 U₁ 和 U₂。若跨模态 equivalence 边 E^equivalence（FCA 证明定义 7.3）将 U₁ 和 U₂ 中的对应点关联，则：
- 同一给定物的两个模态表示收敛到同一社区（或同一社区的不同词元聚合到同一不动点）
- 不同社区的不动点 M* 通过耦合矩阵自然对齐——等价于 FCA 证明定理 7.4 跨模态共现的直接延伸

**公理强制**：P₃——如果"dog"的文本表示和"dog"的图像表示指向同一实体，那么它们的关系模式应等价。提取器提供 equivalence 边，动力学保证等价的输入收敛到近似的不动点（或同一社区内的邻近不动点）。
