# 泛模因理论验证计划

核心假说：在足够丰富的跨领域数据输入下，自适应的结构提取与演化系统将收敛到一组确定的动力学方程。

拆为三个可独立检验的子命题：

| 子命题 | 核心问题 |
|--------|---------|
| **H₁ 收敛性** | 足够丰富的输入下，ODE 系统收敛到确定的动力学模式 |
| **H₂ 统一性** | 不同领域的模因遵循同一组动力学方程 |
| **H₃ 可逆性** | 信息在建模全程不丢失，Φ⁻¹(Φ(I)) ≡ I 在任意规模成立 |

---

## 实验零：收敛判据

在跑任何实验之前，必须把"收敛成功"操作化为可测量的数字。没有这一步，任何实验结果都可以被解释为"参数还没调对"。

```
收敛判据（四条必须同时满足）：
1. 每个模因的 ODE 轨迹在 t_max 内终止于 TerminationReason::Converged
   （而非 MaxSteps 或 NaN）
2. 轨迹末尾连续 5 步的 D/B/ρ/R/S 各自的标准差 < 0.001
3. 分类结果 90% 以上不是 Undetermined
4. 同一个输入数据集在多次独立运行中，原型分布一致
   （同一模因归类到同一原型）

证伪条件：
如果以上四条任意一条在 n ≥ 10⁵ token 的跨领域输入上持续失败，
则 H₁ 被证伪。
```

**实现要求**：收敛判据直接写入 `ode.rs` 的 `TerminationReason` 枚举，新增 `InsufficientConvergence` 变体及对应测试。判据参数（`convergence_window=5`、`std_threshold=0.001`、`undetermined_max=10%`）通过 `OdeConfig` 暴露，可在 CLI 中覆盖。

## 实验一：数据量缩放实验（验证 H₁）

数据量越大，收敛越稳定——这是 H₁ 的核心预测。

### 设计

```
数据: Wikipedia 中文语料（公开免费，CC-BY-SA 许可）
规模: 100 篇 → 1000 篇 → 10000 篇 → 100000 篇
每个规模四次独立运行，跑完整五阶段管线（--text --auto-optimize）
```

|------|--------|--------|---------|---------|-----------|
| 1e2 | | | | | |
| 1e3 | | | | | |
| 1e4 | | | | | |
| 1e5 | | | | | |

### 关键图表

- X = log(数据量)，Y = 收敛率。若 H₁ 为真，曲线应逼近 100%
- X = log(数据量)，Y = 原型分布稳定性。若 H₁ 为真，方差应逐步缩小

### 证伪条件

## 实验二：跨领域一致性实验（验证 H₂）

**最关键实验。** 基因、观念、代码、技术——如果四个完全不同领域的数据在 pan-meme 管线里跑出相同的 ODE 参数分布和原型分布，理论的统一性主张就站得住。如果每个领域各跑各的分布，统一性就是幻觉。

### 数据矩阵

| 领域 | 数据来源 | 规模 |
|------|---------|------|
| 生物演化 | NCBI Taxonomy 物种层级树 + UniProt 蛋白质家族（1000条序列） | ~7MB |
| 技术演化 | arXiv CS 论文摘要（2010-2024），按年份分桶观察术语扩散 | ~500MB |
| 代码演化 | Linux kernel git log 的 commit message，或大型开源项目的 CHANGELOG | ~200MB |
| 观念演化 | Wikipedia "演化"相关词条的编辑历史，观察"自然选择""模因"等词的出现和传播 | ~50MB |

每个领域独立运行完整管线，比较以下四项：

### 比较维度

**a. 五维状态初始分布**：PCA 降维到 2 维画散点图。H₂ 预测四个领域的点云在同一区域，而非分散四角。

**b. ODE 参数分布**：α₁, β₁, ..., ε₂ 各自的直方图。H₂ 预测同一参数在不同领域间大致同分布。

**c. 原型分布**：九类原型的饼图。H₂ 预测原型比例在不同领域间高度相似。

**d. 核心参数跨领域对比表**：

| | α₁ | β₁ | γ₁ | δ₁ | ε₁ |
|------|------|------|------|------|------|
| 生物 | | | | | |
| 技术 | | | | | |
| 代码 | | | | | |
| 观念 | | | | | |
| 均值 | | | | | |
| 标准差 | | | | | |
| 变异系数 | | | | | |

### 证伪条件

- 变异系数 > 30%，统一性不成立

### 成功条件

- 变异系数 < 10%，统一性成立。若达成，这是一篇 Nature 级别的发现——它意味着生物、技术、代码、观念的演化确实由同一组常数驱动，泛模因理论不只是"漂亮框架"，而是描述了客观现实。

## 实验三：可逆性压力测试（验证 H₃）

当前往返验证只做了词数和字数的匹配——这是浅层验证。需要深挖 Φ⁻¹(Φ(I)) ≡ I 在什么条件下会破裂。

### 压力测试矩阵

| 数据类型 | 样本数 | 检验目标 | 预期破裂点 |
|---------|--------|---------|-----------|
| 单字词（猫） | 10 | 字节点冲突 | Phase 1 形式背景退化 |
| 双字词（苹果） | 10 | 组合爆炸 | Phase 2 胞腔复形规模 |
| 三字词 | 10 | 同义异形 | Phase 1 字符 vs 词冲突 |
| 同形异义词 | 10 | 上下文缺失 | Phase 1 无法区分 |
| 随机字符序列 | 10 | 完全无结构 | Phase 1 零收敛 |
| 100% 重复词 | 10 | 退化为单点 | Phase 3 单模因 |

### 具体做法

1. 构造 6 组极端输入，每组 10 个变体
2. 跑完整管线，记录 `max_depth`、Betti 数、五维状态
3. 正向编码 → 反向解码 → 比对恢复率
4. 定位第一个破裂的精确位置：是 Phase 1 的 FCA 闭合失败，还是 Phase 2 的 CW 复形退化？

### 证伪条件

- 任何合法输入（任意中文文本）的恢复率 < 100%。可以排除同形异义等语义层面的破坏——当前 tokenizer 不处理语义，所以同形异义词的"信息丢失"不是理论缺陷，而是输入表征的局限。

### 成功条件

- 所有输入类型恢复率 = 100%，且破裂仅发生在 tokenizer 无法处理的语义歧义上。

## 实验四：消融实验

论文声称四阶段缺一不可。如果去掉一个阶段结果不变，那个阶段就是冗余的。

### 消融矩阵

| 管线变体 | 收敛率 | 原型方差 | 往返恢复率 |
|---------|--------|---------|-----------|
| 完整四阶段（基准） | | | 100% |
| 跳过 Phase 2（几何化） | | | |
| 跳过 Phase 3（分解） | | | |
| 跳过 Phase 4（固化） | | | |
| 只用 Jaccard 不推理 | | | |
| 随机替换 Φ_D/Φ_R | | | |
| 固定参数不用优化器 | | | |

### 成功判断

- 完整管线的收敛率 > 所有消融变体
- 完整管线的原型方差 < 所有消融变体
- 若跳过某个阶段不影响结果，该阶段是冗余的
- 若随机替换函数族也不影响结果，五族函数的区分是徒劳的

## 验收标准

| 等级 | 条件 | 意味着 |
|------|------|--------|
| **证伪** | 实验一收敛率 < 50%，或实验二参数变异系数 > 30%，或可逆性在标准输入上破裂 | 理论的核心主张不成立，需要大幅修订或放弃 |
| **部分验证** | 收敛率 > 50% 但 < 90%，或跨领域参数相似但不显著 | 方向对但预测力不够强，框架有价值但细节需要修正 |
| **验证** | 收敛率 > 90%，实验二跨领域变异系数 < 10%，可逆性 100%，消融确认每阶段必要 | 理论的核心主张成立。立即投 NeurIPS / Nature CompSci |
| **强验证** | 满足"验证"条件，且每个领域至少出现 3 个不同原型（不是全部一个类型） | 不只是收敛，而且区分度好——证明五维空间的信息承载量足够。立即投 Nature |

## 最小可行验证

如果只跑一个实验看答案，从实验一和实验二各取子任务拼成最小可行验证：

```
输入:
  领域 A: 100 篇 Wikipedia 中文文章（生物/演化相关词条）
  领域 B: 100 篇 arXiv NLP 论文摘要（技术/AI相关）

流程:
  --text --auto-optimize，两个领域独立运行

看三个数字:
  1. 收敛率是否 > 50%
  2. 两个领域的 α₁ 差异是否 < 20%
  3. 原型分布是否不完全一样（排除系统只输出一种原型）

如果三天后结果:
  "收敛率 70%，α₁ 差异 12%，领域 A 以基石型为主而领域 B 以爆发型为主"
  → 这个理论值得投一篇顶会。不用等了。
```

### 实现脚本

最小可行验证需要以下脚本：

- `scripts/experiment_mvp.py`：下载 Wikipedia + arXiv 数据 → 分词 → 分领域 → 跑 `updown --text --auto-optimize` → 输出 CSV 对比表
- 数据下载可复用现有 Wikipedia dump 工具（`wikiextractor`）和 arXiv API（`arxiv` Python 包）
- 输出目录：`output/experiments/mvp/`，包含 `biology_summary.csv`、`ai_summary.csv`、`comparison.csv`
