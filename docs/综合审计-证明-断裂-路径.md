## 三、所有可走的路

### 路径 1：继续完善链 A（当前工程主干）

**做法**：修补 v4.0 管线的 8 处断裂，用链 B 的部分成果（如热迹参数映射）替代 from_geometry() 的手工反设计，但保持 FCA + CW + ODE 的大框架不变。

**优点**：
- 已有 Rust 代码可以跑，实验体系已建立
- 外部验证（实验 009）给出初步信号（平均 |r|=0.258）
- 不需要推翻重来

**缺点**：
- CW 复形是装饰（B3 从未被反驳）
- from_geometry() 的反设计本质无法改变——参数不是从结构推导的，是从"想要的结果"反推的
- 缺失原型（Decay/Resilient/Oscillatory/Sink）需要修改 ODE 方程加新项——每一项都是外部假设
- **继续修补意味着你心里清楚你在修补一个概念上有根本问题的框架**

**可行性**：高。你能再发一轮实验 011 把 v4.4 跑出来，再出一篇论文。但这不会解决你的理论焦虑——因为你知道你贴上去的项不是从结构中来的。

### 路径 2：全面转向链 B（谱桥版本）

**做法**：放弃 FCA ↑↓ 循环 + CW 复形 + Betti 分解，用滑动窗口加权邻接矩阵 → 归一化拉普拉斯谱 → 谱社区 → 热迹五维 → 热迹参数 → 谱约束 ODE。

**优点**：
- 数学上最自洽——8 处断裂全部闭合
- 参数从谱/热迹闭式导出（补全 5 的 11 参数公式），零 from_geometry() 反设计
- 五维状态由热迹天然归一化到 [0,1]，不需要外部归一化函数
- 原型分类用 Jacobian 分岔判据（代数条件），非经验阈值
- 代码已有的谱分解、ODE 求解器可以复用

**缺点**：
- **谱 = 信息结构？这是未证明的假设。** 拉普拉斯谱是图论不变量——但一个词网络的拉普拉斯谱到底在什么意义上是"信息-结构"的编码？这从未被论证，它是被"谱是关系模式的代数编码"这个断言跳过去的。
- **热方程 = 信息扩散？这借用了物理隐喻。** 热核 e^{-tL} 描述的是图中随机游走的扩散行为——这和信息结构的演化是同一个东西吗？你可能在用一个物理隐喻替代了信息-结构动力学的本构关系。
- **你失去了 FCA 的哲学根基。** 链 A 中 FCA ↑↓ 循环对应的是"归类-分解"的认知过程——它有认知科学和形式概念分析的学术根基。链 B 从头到尾是图论和偏微分方程——它非常"数学"，但和"认知""演化""模因"的直觉连接变弱了。

**可行性**：高。但你需要面对"我到底是在做信息-结构动力学，还是在做图的热扩散分析"这个问题。

### 路径 3：全面转向链 C（离散 N 迭代版本）

**做法**：放弃 ODE，用离散 N 迭代。P7 直接被翻译为 M^{(k+1)} = N(M^{(k)})，A/(A+B) 的增长/衰减竞争形式。收敛到不动点。输出信息作用量谱。

**优点**：
- 真正"零自由参数"——A/(A+B) 是被 P1+P3+P5+P7+P8 五条公理唯一强制的形式
- 不借任何物理隐喻——不需要热方程、不需要 ODE、不需要 smooth flow
- 与 P7（复制-变异-选择）的离散本质一致——信息演化的每一"步"都是离散事件
- 很干净的信息作用量谱作为输出——五个分量对应 P2 三条件

**缺点**：
- 工程上几乎从零开始——你现有的 Rust 代码（ODE 求解器、ODE 分类器）全部作废
- 11 参数闭式映射未给出（目前只声明"由谱参数决定"）
- Jacobian 压缩性的解析证明缺失（目前仅有数值佐证）
- 输出是作用量谱而非原型分类——论文的实验体系需要重新设计
- **A/(A+B) 虽然由公理强制出来很美，但它在数学上是压缩映射这件事需要证明，目前只论证了"选择算子应该压缩"**

**可行性**：中。需要重写大部分代码，但理论框架是最干净的。代价是失去已有实验积累。

### 路径 4：接受断裂，走"建模选择 + 验证驱动"路线

**做法**：诚实标注每一步是"数学强制"还是"建模选择"。ODE 形式是建模选择，参数映射是建模选择，原型分类也是建模选择。不声称"每一步由公理唯一决定"。然后用实验 008-010 的验证结果（收敛率 100%、外部验证通过）作为"虽然这些是建模选择，但选择的组合产生了可验证的预测"的证据。论文定位为"一个可证伪的假说"而非"一套完备的数学理论"。

**优点**：
- 诚实——不强称每一步都是公理驱动的唯一解
- 保留了目前所有的代码和实验积累
- 同行评审不会因为"你声称从公理推导但实际不是"而拒稿
- 这正是你论文摘要里写的——"若收敛失败则该假说被证伪"。你一开始就没承诺"完备推导"

**缺点**：
- 失去了"零自由度"的叙事——参数映射是人为的、ODE 形式是人为的
- 但你用实验证明这些人为选择产生了可验证的预测
- 这本质上是"先建模、再验证"的工程路径，不是"从第一原理推导"的数学路径

**可行性**：最高。但你需要和自己和解——你不能再同时声称"每一步被公理强制"和"这些都是建模选择"。两个叙事不能共存。

### 路径 5：回到 FCA，往下走范畴论

**做法**：你已经在 FCA 阶段有了完备的概念格。概念格是完备格——它天然是一个范畴：对象是形式概念，态射是子概念-超概念的偏序关系 ⪯。与其跳到 CW 复形（B3 是装饰）或拉普拉斯谱（热方程是借来的），不如顺着概念格的结构往下走：

- 概念格 → 层（sheaf）——一个形式概念上的层给每个子概念赋一个"局部数据"，胶合条件对应概念格中的上下因果
- ⪯ 偏序 → 前层（presheaf）——从概念格到任意目标范畴的函子
- 层上导出函子 → 上同调 → 结构不变量
- 不变量 → 分类

**优点**：
- 完全在 FCA 内部——没有外部物理隐喻
- 范畴论天然处理"层级涌现"（P8）和"结构不变性"（P3）
- 上同调不变量是真正从结构中涌现的——不需要"选"函数族或"贴"参数
- Grothendieck 拓扑的理论基础已经完备

**缺点**：
- 你需要同时学会或补上：格论、范畴论、层论、导出函子、上同调——这可能是一个博士级别的数学投入
- 工程上从零开始
- 目前地球上可能没有人做过"FCA + 层论 + 信息动力学"的组合——你是第一个

**可行性**：低（从工程角度）。但从"你心里想要什么"的角度，这可能是唯一一条真正不借外部隐喻的路。**概念格是 FCA 给你的，层是概念格上自然长出来的——全程没有借任何东西。**

### 路径 6：随机过程替代 ODE

**做法**：P7 说的是"复制-变异-选择"。在生物学里，这个过程的数学语言从来不是 ODE——是分支过程（branching process）、Moran 模型、Wright-Fisher 模型。ODE 是"大种群极限"下的近似——而你的社区规模（词数、概念数）远不到大种群极限。用随机过程替代 ODE：

- 每个社区的状态不是 [0,1]^5 的连续轨迹，而是计数过程（community size、边数、谱隙的离散时间序列）
- "复制" = 种群增长，"变异" = 谱结构漂移，"选择" = 基于适应度的保留/淘汰
- 输出 = 过程的遍历性质、吸收态、灭绝概率
